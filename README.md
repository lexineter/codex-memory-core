## 1. Project Overview
Codex Memory Core is deterministic AI memory infrastructure designed for replay-verifiable state evolution. Every accepted event is encoded deterministically, committed cryptographically, and replay-checked against the exact same rules.

Replay-verifiable memory matters because it turns history into proof. Instead of trusting operational logs or wall-clock narratives, a verifier can reconstruct state from bytes and deterministically detect tampering, drift, or rule violations.

## 2. Key Properties
- Deterministic state evolution across inserts, scoring, lifecycle transitions, snapshots, divergence locators, and snapshot deltas.
- Cryptographic commitments for event payloads, authenticated roots, query/scoring paths, lifecycle transitions, and snapshot state views.
- Replay verification that re-derives commitments and state transitions from ledger bytes and fails closed on any mismatch.
- Governance enforcement for lifecycle mutation flows under declared feature flags and deterministic transition rules.
- Proof bundle artifacts for reproducible shipping, verification, and audit-ready review.

## 3. Architecture Overview
- `codex_core`: deterministic protocol engine, schema, ledger I/O, replay verifier, proofs, and artifact tooling.
- `codex_cli`: external command surface for replay verification, bundle generation, demo execution, and version reporting.
- Deterministic ledger: append-only binary records with strict framing and domain-separated commitments.
- Snapshot commitment: convergence anchors over replay state and authenticated roots, with optional document-level Merkle state.
- Proof bundle: reproducible artifact package generated by `ship.sh`/`release_bundle.sh` and verified by `verify_bundle.sh`.

## 4. Quickstart
Run the full deterministic packaging and verification path:

```bash
./scripts/ship.sh
```

Expected terminal markers include:

```text
DETERMINISTIC_PROOF_CONFIRMED
BUNDLE_VERIFIED
SHIP_READY:dist/proof_bundle
```

## 5. Verification
Replay-verify the produced ledger:

```bash
cargo run -q -p codex_cli -- replay --ledger dist/proof_bundle/proof_a/ledger.bin
```

## 6. Container Usage
Build and run the containerized CLI:

```bash
docker build -t codex-memory .
docker run codex-memory version
```

## 7. Benchmarks
Generate deterministic benchmark output:

```bash
./scripts/bench.sh
```

Output is written to `dist/benchmarks.json`.
